     2. Implementation Phases

     Phase 1: Foundation (plugin manifest, CLAUDE.md, core data)

     Files to create:

     1. .claude-plugin/plugin.json — Plugin manifest with name "lesson-agent",
     version "0.1.0", description, keywords (education, syllabus, accreditation,
      career-alignment)
     2. CLAUDE.md — Agent identity document establishing:
       - Agent name and purpose
       - Two persona modes: Faculty (full capabilities) and Student (read-only
     recommendations, no syllabus modification)
       - Safety principles: never bypass accreditation guardrails, always check
     constraints before recommending, be transparent about data limitations
       - Data locations: where to find user profiles, syllabi, constraints,
     accreditation data
       - Workflow order: analyze -> align -> constrain -> recommend -> validate
       - Privacy rules: student data shared with faculty only in
     aggregate/anonymized form
     3. data/ directory — All example user profiles, syllabi, accreditation
     data, and industry skills data (detailed in Section 5 below)

     Phase 2: Skills (6 skill definitions with references)

     Each skill follows the SKILL.md pattern with YAML frontmatter (name,
     description with third-person trigger phrases) and markdown body (1500-2000
      words), with detailed content in references/ subdirectories.

     Skill 1: skills/syllabus-analysis/SKILL.md
     - Description triggers: "analyze a syllabus", "audit course materials",
     "parse a syllabus", "check learning outcomes", "review course objectives"
     - Core workflow: (1) Parse syllabus sections, (2) Classify learning
     outcomes by Bloom's taxonomy, (3) Map topics to weeks, (4) Cross-reference
     against accreditation standards, (5) Produce structured audit report
     - References: bloom-taxonomy.md (6 levels with verb lists),
     syllabus-parsing-patterns.md (extraction heuristics for different document
     formats)

     Skill 2: skills/goal-alignment/SKILL.md
     - Description triggers: "match career goals to courses", "align student
     goals with curriculum", "find career-relevant content", "identify skill
     gaps"
     - Core workflow: (1) Accept career goals, (2) Normalize against career
     taxonomy, (3) Extract required competencies, (4) Compare against course
     learning outcomes, (5) Score alignment (strong/partial/missing), (6)
     Produce alignment matrix
     - References: career-taxonomy.md (O*NET-derived clusters),
     industry-skills-mapping.md (tech, business, engineering skill ladders)

     Skill 3: skills/recommendation-generation/SKILL.md
     - Description triggers: "generate course recommendations", "suggest
     syllabus changes", "modernize a course", "create a revision plan"
     - Core workflow: Generate tiered revision plans — Tier 1 (quick wins, <2
     hrs each), Tier 2 (moderate, 5-20 hrs), Tier 3 (deep restructuring, 40+
     hrs). Each recommendation annotated with effort estimate, budget impact, TA
      support needed, accreditation safety flag
     - References: revision-tiers.md (tier definitions with examples),
     workload-estimation.md (effort calculation methodology)

     Skill 4: skills/constraint-management/SKILL.md
     - Description triggers: "set constraints", "check workload limits", "manage
      budget constraints", "verify TA availability", "check feasibility"
     - Core workflow: Accept faculty constraints (hours, budget, TA capacity,
     timeline, department approval), validate proposed changes against them,
     compute feasibility score (0-100), flag violations, suggest alternatives
     for infeasible changes
     - References: constraint-types.md (all constraint types with scoring
     rubrics)

     Skill 5: skills/assessment-redesign/SKILL.md
     - Description triggers: "redesign an assessment", "create alternative
     assessments", "design a career-relevant assignment", "build a rubric"
     - Core workflow: (1) Identify original assessment's learning outcomes, (2)
     For each target career path identify authentic tasks, (3) Design
     career-branched assessment variants, (4) Create rubrics mapping to original
      learning outcomes, (5) Estimate workload parity
     - References: assessment-types.md (authentic assessment catalog),
     rubric-templates.md (structured rubric format with criteria, performance
     levels, career-skill annotations)

     Skill 6: skills/accreditation-guardrails/SKILL.md
     - Description triggers: "check accreditation compliance", "verify changes
     meet ABET requirements", "validate curriculum changes against degree
     requirements"
     - Core workflow: For each proposed change, check (1) learning outcome
     coverage preserved, (2) contact hours maintained, (3) assessment coverage
     adequate, (4) core topics not removed without replacement. Produce
     pass/warning/fail for each standard. Escalation protocol for violations.
     - References: abet-criteria.md (ABET CAC student outcomes SO-1 through
     SO-6), general-education-requirements.md (UMaine gen-ed WOC, QL, ETH, CTCE)

     4. Agent Design Principles

     These principles govern all skills and interactions:

     1. Guardrails over alignment — Never remove foundational content or violate
      accreditation, even when industry data superficially supports it
     2. Constraints first — Inventory faculty constraints before generating any
     recommendations
     3. Aggregate, not individual — Student career data shared with faculty only
      in aggregate/anonymized form
     4. Format over substance — Modify assessment formats and topics for career
     alignment without changing learning outcomes or reducing rigor
     5. Honest about limitations — When data is insufficient, say so and provide
      a path to fill the gap rather than hallucinating
